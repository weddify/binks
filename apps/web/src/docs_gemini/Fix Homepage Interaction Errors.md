The color scheme is defined in **[apps/web/tailwind.config.ts](cci:7://file:///e:/web-apps/binks/apps/web/tailwind.config.ts:0:0-0:0)**.

We are using a hybrid approach combining **DaisyUI Themes** and **Custom Tailwind Colors**:

### 1. DaisyUI Themes (The Foundation)
In [tailwind.config.ts](cci:7://file:///e:/web-apps/binks/apps/web/tailwind.config.ts:0:0-0:0), we load two preset themes from DaisyUI:
```typescript
daisyui: {
  themes: ["cupcake", "sunset"], // cupcake = light, sunset = dark
  darkTheme: "sunset",
},
```
*   **"cupcake"**: Provides the default light pastel colors.
*   **"sunset"**: Provides the default dark elegant colors.

### 2. Custom Overrides
In the same file, we extend the theme with specific named colors that you see used in the code (like `bg-background-light`):
```typescript
theme: {
  extend: {
    colors: {
      primary: "#4a6df7",           // Your brand blue
      "primary-dark": "#3b5bdb",
      "background-light": "#f5f6f8", // Light mode background
      "background-dark": "#101322",  // Dark mode background
      "surface-light": "#ffffff",
      "surface-dark": "#1e2330",
    },
    // ...
  },
}
```

### How it works:
*   **Logic**: [src/lib/components/ThemeToggle.svelte](cci:7://file:///e:/web-apps/binks/apps/web/src/lib/components/ThemeToggle.svelte:0:0-0:0) switches the `data-theme` attribute on the `<html>` tag between "cupcake" and "sunset".
*   **Usage**: When you write `btn-primary`, it uses the color from the active DaisyUI theme (or our override if we mapped it). When you write `bg-background-light`, it uses your custom specific hex code.
